# Mac Text Actions 设计文档

## 1. 产品形态
应用采用“后台常驻 + `global shortcut` + 轻量 `result panel`”的形式。主流程不依赖完整主窗口，也不依赖菜单栏操作。

菜单栏承担轻量控制与辅助职责：
- 切换当前默认模式
- 展示模式对应快捷键
- 打开设置
- 退出应用

## 2. 主流程设计
1. 用户在任意应用中选中一段文本
2. 用户按下 `global shortcut`
3. 应用优先读取当前 `selected text`
4. 若读取失败，自动尝试 `clipboard fallback`
5. 应用按固定优先级识别内容类型
6. 对低歧义类型直接计算 `primary result`
7. `result panel` 展示结果、原文摘要、内容来源和 `secondary action`
8. 用户决定复制结果、替换原文、执行附加动作或关闭

新增模式切换路径：
1. 用户点击状态栏菜单
2. 用户在单层菜单中选择 `自动识别` 或某个具体功能模式
3. 若菜单处于展开状态，用户也可按 `⌘1` 到 `⌘7` 切换默认模式
4. 用户在任意应用中按下 `global shortcut`
5. 应用按当前默认模式执行，并展示对应的 `result panel`

## 3. 识别优先级
识别优先级在全产品中固定如下，并作为唯一权威定义：

1. 合法 `JSON`
2. `10` 位或 `13` 位纯数字时间戳
3. 可解析日期字符串
4. 普通文本

其他文档引用本节，不重复定义规则，以避免漂移。

## 4. 默认行为
### 4.1 自动生成 `primary result`
- `JSON`：直接展示格式化结果
- `Timestamp`：直接展示本地日期时间
- `Date String`：直接展示时间戳

### 4.2 不自动执行的能力
- `JSON Compress`
- `MD5`
- `Create Reminder`
- `Copy Result`
- `Replace Selection`

这些能力统一作为 `secondary action` 暴露。

### 4.3 指定模式执行
状态栏菜单可把以下能力设置为默认模式：
- `自动识别`
- `JSON 格式化`
- `JSON Compress`
- `时间戳转本地时间`
- `日期转时间戳`
- `MD5`
- `创建提醒事项`

当默认模式不是 `自动识别` 时，`global shortcut` 直接按该模式执行，而不是再次参与自动识别分流。

## 5. 结果面板行为
`result panel` 必须满足以下原则：
- 触发后有明确反馈
- 自动识别后直接渲染结果，不强迫用户先选工具
- 不在用户无确认的情况下改写原文
- 同时保留继续动作的入口

面板区块建议为：
- 顶部：执行来源标题，如 `自动识别 · JSON` 或 `指定模式 · MD5`，以及原文摘要
- 顶部辅助信息：当前内容来源，如 `selected text` 或 `clipboard fallback`
- 中部：`primary result` 或错误信息
- 底部：与类型相关的 `secondary action`

## 6. 错误处理
- 无 `selected text` 且剪贴板无可用文本：显示 `未检测到可处理文本`
- 当前应用不支持读取选区但剪贴板有可用文本：继续主流程，并显示 `已改用剪贴板内容`
- 当前应用不支持读取选区且剪贴板无可用文本：显示 `当前应用暂不支持读取选中文本`
- 非法 `JSON`：显示 `JSON 校验失败` 及错误说明，不降级为普通文本
- 日期无法解析：按普通文本处理，不展示错误
- 替换失败：保留当前结果，并提示改为复制

## 7. 设计原则
- 默认路径只覆盖低歧义转换
- 工具型能力必须显式触发
- 错误反馈优先于静默容错
- UI 足够轻量，避免打断用户当前上下文
- 能力按类型聚合，而不是按工具列表堆砌

## 8. 剪贴板回退保护规则
- `clipboard fallback` 在直接读取 `selected text` 失败后自动触发一次复制回退
- 若复制后剪贴板仍无可用文本，则视为回退失败
- 发生 `clipboard fallback` 时，`result panel` 必须明确提示该内容来自剪贴板回退，而不是直接读取到的实时选区
