# Mac Text Actions 方案文档

## 1. 方案目标
为 macOS 提供一个优先围绕 `selected text`、并在读取失败时支持 `clipboard fallback` 的统一文本动作入口。方案需要兼顾主流程效率、识别准确性、系统权限约束和后续扩展性。

## 2. 备选方案
### 方案 A：纯菜单栏工具
通过菜单栏打开工具窗口，再手动粘贴或读取当前内容。

优点：
- 结构直观
- 实现风险较低

缺点：
- 不是“选中即处理”
- 入口路径长
- 用户心智更像普通工具集合

### 方案 B：`global shortcut` 主导 + 轻量 `result panel`
后台常驻监听 `global shortcut`，读取 `selected text`，识别后直接渲染 `primary result`，再通过 `result panel` 提供 `secondary action`。

优点：
- 最贴合核心需求
- 路径短
- 自动结果与显式动作边界清晰

缺点：
- 需要处理系统权限
- 依赖前台应用的选区可读性

### 方案 C：识别后直接执行并写回
快捷键触发后自动完成识别、转换和原文替换。

优点：
- 极快

缺点：
- 误判成本高
- 缺乏可控性
- 不适合作为 v1 默认体验

## 3. 结论
采用方案 B。

原因如下：
- 满足“选中即处理”的核心目标
- 自动行为被限定在低歧义结果展示，不会偷偷写回
- 方便后续增加更多 `secondary action`
- 与 macOS 工具型应用的使用心智一致

## 4. 关键约束
- 优先处理当前 `selected text`
- 仅在读取失败时允许 `clipboard fallback`
- 发生 `clipboard fallback` 时必须明确标注来源
- 不自动写回原文
- `JSON` 默认只做格式化
- `MD5` 只能作为 `secondary action`

## 5. 分阶段落地建议
### v1
- `global shortcut`
- `selected text` 读取
- `JSON` 格式化/校验
- 时间戳与日期互转
- `MD5`
- 快速创建提醒事项
- `result panel`
- `Copy Result` / `Replace Selection`

### v1.1+
- 新增更多文本类型
- 支持更丰富的结果视图
- 支持用户自定义默认动作

## 6. 风险与缓解
### 6.1 无法读取选区
风险：部分应用无法稳定提供 `selected text`。  
缓解：优先读取选区，失败后尝试 `clipboard fallback`，并在界面中明确标注内容来源；若剪贴板也无可用文本，再给出明确错误提示。

### 6.2 日期字符串歧义
风险：不同格式存在解析不一致。  
缓解：只对明确可解析格式生成 `primary result`，否则按普通文本处理。

### 6.3 替换原文失败
风险：焦点变化或权限限制导致替换失败。  
缓解：保留结果并提示复制。

### 6.4 提醒事项权限
风险：系统权限不足导致创建失败。  
缓解：首次引导授权，并在设置中显示权限状态。
