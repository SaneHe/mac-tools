# 复制反馈 HUD 组件设计

## 背景

当前仓库中的复制反馈存在两套割裂实现：

- `result panel` 与独立工具窗口中的复制按钮只负责写入剪贴板，没有统一的成功反馈层。
- `SelectableCopyableText` 与 `CenteredTextView` 内部使用基于 `AppKit` 的黑色 `toast`，视觉风格偏旧，也无法与上层 `SwiftUI` 容器共享状态。

这会带来两个问题：

- 不同复制入口的反馈不一致，用户难以建立稳定预期。
- 复制动作与反馈展示耦合在各自入口里，不利于后续复用到“已替换”“已创建提醒事项”等同一家族的状态确认组件。

## 目标

为所有复制入口提供一套统一、克制、带有轻微苹果风格动效的“已复制”反馈组件，并同时覆盖：

- `result panel` 底部复制按钮
- 独立工具窗口中的复制按钮
- 输出文本区双击复制

这套组件需要遵守当前仓库的 UI 方向：

- `Native macOS utility panel + refined polish`
- 轻量、克制、立即反馈
- 不抢占 `primary result`
- 对 `macOS 13 Ventura` 保持兼容

## 非目标

- 不新增新的复制入口
- 不修改复制动作本身的业务语义
- 不把复制反馈扩展成全局通知系统
- 不引入只适用于新系统的重度 Liquid Glass API 依赖

## 设计选项

### 方案一：Monochrome HUD

- 形态：底部居中的中性色小胶囊
- 动效：轻微上浮与淡入淡出
- 优点：实现稳妥，最容易兼容 `macOS 13`
- 缺点：苹果感偏弱，容易只像“旧 toast 的精修版”

### 方案二：Liquid Glass HUD（推荐）

- 形态：底部居中的浅玻璃感胶囊，带极轻的高光、描边与阴影
- 动效：短促弹入、轻微 settle、停留后淡出
- 优点：最贴近当前 `result panel` 与 `LiquidGlassPopover` 的质感方向，也最符合“带一点苹果风格小动画”的目标
- 风险：如果材质过重，会抢结果内容；实现上必须用对 `macOS 13` 友好的材质近似，而不是依赖过新的专有 API

### 方案三：Micro Banner

- 形态：贴近动作栏上方的一条窄状态条
- 动效：从动作栏中抽出再回落
- 优点：与按钮动作关联更强
- 缺点：不适合双击复制这类非按钮入口，也不够通用

## 设计结论

采用 `方案二：Liquid Glass HUD`，但控制为“克制版”：

- 使用浅玻璃感、轻描边和低强度阴影表达系统感
- 不做强颜色强调，不使用庆祝式成功动画
- 统一固定文案 `已复制`
- 所有复制入口共用同一套视觉与时序规则

## 视觉设计

### 形态

- 底部居中小胶囊
- 高度控制在 `30~34pt`
- 宽度自适应内容，并设置舒适的最小宽度
- 胶囊圆角偏大，接近系统工具型 HUD，而不是通知卡片

### 内容

- 左侧使用 `checkmark`
- 右侧固定文案 `已复制`
- 图标与文字间距紧凑，整体居中排列
- 图标亮度略高于文字，但不做绿色成功态

### 材质

- 使用对 `macOS 13` 友好的半透明浅材质近似玻璃感
- 配合极轻描边帮助 HUD 在深浅内容上都能站住
- 保留一层很轻的浮起阴影，避免沉重感

### 位置

- 始终贴容器底部居中显示
- 与容器底边保留 `16~20pt` 间距
- 不贴按钮本体，不直接压在 `primary result` 上

## 动效设计

动效遵守“短促、安静、确认感强”的原则，分为三段：

### 入场

- `opacity` 从 `0` 到 `1`
- `y` 轴从下向上移动 `4~6pt`
- 时长控制在 `120~160ms`

### settle

- 使用很轻的 `spring` 或缩放回弹
- 整体从 `0.98 -> 1.0`，或 `1.02 -> 1.0`
- 时长控制在 `180~240ms`

### 退场

- 停留 `0.9~1.1s` 后淡出
- 不做横向移动或过度收缩
- 时长控制在 `180~220ms`

### 重复触发

- 同一容器内不堆叠多个 HUD
- 若 HUD 仍在显示，则刷新可见时长并轻微重播入场动画

## 组件结构

建议拆成两层：

### `CopyFeedbackHUD`

纯展示组件，只负责绘制 HUD：

- 输入：`isVisible`、动画重播标识、底部 inset
- 输出：无
- 不直接操作剪贴板
- 不关心触发来源

### `CopyFeedbackHost`

承载状态与展示时序的容器层：

- 维护 HUD 的显隐状态
- 处理自动隐藏
- 处理重复触发时的计时刷新与轻微重播
- 作为 `SwiftUI` 视图的 overlay 宿主

## 状态流

统一复制反馈流程如下：

1. 用户从任一复制入口触发复制
2. 复制动作先真正写入剪贴板
3. 写入成功后，向所在容器发送“复制成功”事件
4. `CopyFeedbackHost` 显示 HUD
5. 若短时间内再次触发，则重置 HUD 可见时间并轻微重播动画
6. 定时结束后自动淡出

## 接入边界

### `result panel`

- HUD 作为整个 `LiquidGlassPopover` 的底部 overlay
- 复制成功后先展示 HUD，再短暂延迟关闭 `popover`
- 推荐延迟时长约 `0.65s`，既保证反馈可见，又不让面板停留过久

### 独立工具窗口

- HUD 作为 `ToolWorkspaceView` 根层的底部 overlay
- 复制成功后不改变当前窗口状态，仅展示 HUD

### 双击复制

- `SelectableCopyableText` 与 `CenteredTextView` 不再自行绘制旧式 `AppKit toast`
- 文本视图只负责完成复制并向上抛出成功事件
- HUD 统一由外层 `SwiftUI` 容器展示

## 错误与边界处理

- 无可复制内容时，不显示 HUD
- 剪贴板写入失败时，不显示 `已复制`
- HUD 不排队，不叠层
- 容器销毁时应取消挂起的隐藏任务，避免孤儿动画

## 测试要求

### 状态层测试

- 单次触发后进入可见态
- 连续触发时只刷新状态，不新增实例
- 到时后自动隐藏

### 视图布局测试

- HUD 在 `result panel` 与工具页中都位于底部居中
- 不遮挡主要结果内容
- 在不同结果宽度下保持稳定

### 行为测试

- `result panel` 复制按钮成功后能显示 HUD，并按预期延迟关闭
- 工具页复制按钮成功后能显示 HUD
- 文本区双击复制成功后，能通过外层容器显示 HUD

## 实现约束

- 视觉上接近现代 macOS 材质，但实现上优先保证 `macOS 13` 兼容
- 反馈展示统一收口到 `SwiftUI` 容器层
- `AppKit` 文本视图保留最小职责，只做复制与事件回传
- 维持当前“轻量浮层工具”的产品气质，不引入通知中心式沉重交互

## 后续扩展

如果本次 HUD 结构稳定，后续可以复用同一套组件族承载：

- `已替换`
- `已创建提醒事项`
- `已切换为剪贴板回退`

但这些扩展不属于本次范围，本次只落地固定文案 `已复制`。
