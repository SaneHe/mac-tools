# MD5 优先级与顶部菜单收缩设计

## 背景

当前 `Mac Text Actions` 的顶部状态栏菜单同时承担了两类职责：

- 自动识别已可稳定命中的能力入口
- 用户希望强制执行的显式模式入口

这会带来两个问题：

- 菜单项过多，默认模式列表与自动识别能力重复
- `MD5` 作为普通文本场景下更常见的显式动作，在菜单中的优先级不够直观

同时，现有自动识别流程对 `plain text` 仅返回 `actionsOnly`，但在 `result panel` 中缺少足够明确的推荐提示，无法凸显 `MD5` 作为默认下一步动作的定位。

## 设计目标

- 保持自动识别优先级不变：`JSON -> 时间戳 -> 日期字符串 -> 普通文本`
- 普通文本仍不自动生成 `primary result`
- 将 `MD5` 作为普通文本场景下最突出的推荐动作
- 收缩顶部状态栏菜单，只保留真正需要“显式指定模式”的入口
- 同步更新快捷键、入口文案与文档，避免菜单职责漂移

## 设计结论

### 1. 顶部菜单只保留 3 个默认模式

状态栏菜单模式顺序调整为：

1. `自动识别`
2. `MD5`
3. `JSON Compress`

移除以下模式入口：

- `JSON 格式化`
- `时间戳转本地时间`
- `日期转时间戳`

原因：

- 这些能力已可被自动识别稳定命中，并直接产生低歧义结果
- 顶部菜单不再承担“能力总表”角色，而是只保留默认主流程模式

### 2. 菜单快捷键同步收缩

菜单展开时的模式切换快捷键同步调整为：

- `自动识别 = ⌘1`
- `MD5 = ⌘2`
- `JSON Compress = ⌘3`

不再保留 `⌘4` 到 `⌘6` 的模式切换说明。

### 3. 普通文本仍不自动执行，但强化 MD5 推荐

自动识别命中 `plain text` 时：

- 仍保持 `actionsOnly`
- 不直接生成 `MD5` 结果
- 在 `result panel` 中显示一条短提示，说明当前未命中自动转换类型
- 将 `MD5` 作为最突出的推荐动作放在动作栏首位

推荐提示文案采用保守表达，避免让用户误解为系统已经执行了转换。

推荐文案方向：

- 标题：`未识别为 JSON 或时间类型`
- 辅助说明：`可以继续执行 MD5 或其他文本动作`

### 4. secondary action 排序规则

对于 `plain text` 的 `actionsOnly` 结果，底部动作顺序调整为：

1. `MD5`
2. 其他文本动作

本次设计仅明确 `MD5` 的优先级。当前实现中若仍保留 URL 或提醒事项相关动作，可继续展示，但它们的视觉权重与顺序都应低于 `MD5`。

### 5. 结果态保持既有 MD5 option 规则

当用户点击 `MD5` 后：

- 进入 `MD5` 结果态
- 默认输出小写摘要
- 底部继续使用 `转大写 / 转小写` option 按钮

这部分不改变当前已落地的 option 型 `secondary action` 规则。

## 影响范围

本次设计将同步影响：

- `README.md`
- `docs/README.md`
- `docs/product.md`
- `docs/ui/mac-text-actions-ui.md`
- 状态栏菜单与设置页快捷键摘要
- 自动识别的普通文本结果面板交互

## 不纳入本次改动的内容

- 不调整自动识别优先级
- 不让普通文本自动直接生成 `MD5`
- 不重新定义 URL 相关能力是否属于正式 v1 范围
- 不扩展新的顶部菜单模式
