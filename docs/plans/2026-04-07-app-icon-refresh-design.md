# Mac Text Actions 图标重设计方案

## 1. 背景
- 当前应用图标由运行时代码生成，已接入应用启动流程，可直接通过重绘逻辑完成替换
- 现有图标语义偏向“循环处理”，与产品“对 `selected text` 执行动作”的核心场景不够贴合
- 仓库现有文档已经约束图标核心语义为 `text cursor + sparkles`

## 2. 目标
- 保留 `Native macOS utility panel + refined polish` 的系统工具气质
- 让图标更直接表达“选中文本 + 转换动作”的产品含义
- 保持状态栏图标可简化为单色模板图，确保菜单栏场景识别度
- 不引入资源目录改造，继续复用当前代码生成图标的构建链路

## 3. 最终选择
用户确认采用 `B` 方案：`功能隐喻感`

### 3.1 应用图标结构
- 背景：蓝色渐变圆角方块，顶部带轻微高光层，延续轻量工具属性
- 主体：两层错位文本卡片，表达“当前文本内容”和“处理后的文本语境”
- 核心符号：居中的 `text cursor`，作为语义锚点
- 动作提示：右上区域使用暖色 `sparkles`，表达“执行动作 / 变换 / 增强”
- 视觉重点：以深蓝 `cursor` 和暖黄 `sparkles` 形成主对比，避免继续使用不相关的循环箭头或徽章结构

### 3.2 状态栏图标结构
- 使用简化后的单色模板图
- 保留 `text cursor` 作为主轮廓
- 以一个较小的火花作为辅助特征，避免菜单栏里退化成普通文本编辑图标
- 线条与形状保持简洁，优先保证小尺寸可辨识度

## 4. 落地范围
- 修改 `Sources/MacTextActionsApp/AppIconFactory.swift`
- 修改 `Tests/MacTextActionsAppTests/AppIconFactoryTests.swift`
- 新增本设计记录与对应实现计划文档

## 5. 验收标准
- 应用启动后可正常设置 Dock 图标，无运行时报错
- 状态栏图标仍以模板图方式渲染
- 图标在 `512x512` 和小尺寸状态栏场景下均保持基本识别度
- 测试能够覆盖图标尺寸、模板渲染以及关键视觉特征
